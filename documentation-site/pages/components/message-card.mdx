# Message Card

<Yard placeholderHeight={52} {...messageCardYardConfig} />

A message card is a snapshot-like preview of a message intended to encourage users to click or tap to view more details.
It is made up of a heading, a paragraph, an image asset, and a CTA (styled as a button). All parts of the message card can be configured on and off to suit the needs of the message.

## When to use

- Engagement messaging
- Promotions
- Brand campaigns
- Partnerships
- Upselling a new feature

Not recommended for:

- Global system messaging (use a Banner instead)
- Ephemeral feedback messaging (use a Snackbar instead)

</Example>

Message cards leverage all the colors in the Base Web primitive color palette.

To ensure text is always legible, MessageCard determines the appropriate text color based on the background color.

Unlike UI components that “flip” in dark mode, cards with rich illustrations can’t simply be “flipped”. The result often looks more like a photo negative. Instead of flipping colors in dark mode, MessageCard uses an overlay to dim the background color and image so the colors don’t vibrate too much in a dark environment. Note that the overlay is below the text layer to ensure accessible contrast ratios.

When using a color outside of the Base Web primitive color pallete, use the `backgroundColorType` prop to indicate whether the color is light or dark. MessageCard will determine the appropriate font color accordingly.

Images can either be placed at the top of the card or at the trailing end. For cards with trailing images, the image container width is fixed to a default size of 112. The height of the image will depend on the amount of content placed inside the card.
